Task 1: Data Binding
1. Interpolation
   Syntax: {{ property }}
   Purpose: Displays component data in the template.
   Example:
   {{ portalName }}
2. Property Binding
   Syntax: [property]="expression"
   Purpose: Sends data from component to DOM.
   Example:
   [disabled]="!isPortalActive"
3. Event Binding
   Syntax: (event)="method()"
   Purpose: Handles user actions.
   Example:
   (click)="onEnrollClick()"
4. Two-Way Binding
   Syntax: [(ngModel)]="property"
   Purpose: Synchronizes component and UI data.
   Example:
   [(ngModel)]="searchTerm"
Difference:
[property] → One-way binding (Component → DOM)
[(ngModel)] → Two-way binding (Component ↔ DOM)
Task 2: Lifecycle Hooks
1. ngOnInit()
   Fires once when component is initialized.
   Used for loading data.
   Output:
   HomeComponent initialised — courses loaded
2. ngOnDestroy()
   Fires when component is removed from the page.
   Used for cleanup and unsubscribing.
   Output:
   HomeComponent destroyed
3. ngOnChanges()
   Fires whenever an @Input() property changes.
   Example:
   @Input() course: any
   Output:
   Previous Value: undefined
   Current Value: {name: 'Angular', code: 'ANG101', credits: 4}
Component Communication:
@Input()
Used to pass data from parent component to child component.
Example:
CourseListComponent → CourseCardComponent
